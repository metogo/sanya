# 快速测试 FilterBar Sticky 功能

## 🚀 快速开始

1. **启动服务器**
   ```bash
   npm run dev
   ```

2. **打开浏览器**
   访问 `http://localhost:3000`

3. **测试步骤**
   - 页面加载后，观察 FilterBar（应该是正常大小）
   - 向下滚动页面
   - 观察 FilterBar 是否：
     - ✅ 变小（按钮变小、标签消失）
     - ✅ 固定在屏幕上（不随滚动消失）
     - ✅ 右上角出现绿色 "Mini Mode" 标签
   - 向上滚动回顶部
   - 观察 FilterBar 是否恢复正常大小

## 🐛 如果没有效果

### 检查 1: 页面是否可以滚动？
- 如果页面内容不够长，无法滚动，功能不会触发
- 解决：添加更多景点数据，或缩小浏览器窗口高度

### 检查 2: 浏览器控制台有错误吗？
- 按 F12 打开开发者工具
- 查看 Console 标签是否有红色错误
- 如果有错误，请截图反馈

### 检查 3: 是否看到绿色 "Mini Mode" 标签？
- 如果滚动后看到绿色标签，说明状态切换成功
- 如果没看到，说明滚动检测可能有问题

## 🔧 调整建议

如果 FilterBar 被 Header 遮挡，可以调整 `top` 值：

打开 `src/components/FilterBar.tsx`，找到第 77 行：
```tsx
isCompact ? 'py-2 md:py-3 z-40 top-[100px] md:top-[120px]' : 'py-6 md:py-8 z-30 top-0'
```

调整 `top-[100px]` 和 `top-[120px]` 的值，直到 FilterBar 正好在 Header 下方。

## 📸 预期效果

**正常模式（页面顶部）**：
- 大按钮
- 有 "CATEGORIES"、"RATINGS"、"PRICES" 标签
- 充足的间距

**紧凑模式（滚动后）**：
- 小按钮
- 没有标签文字
- 紧凑的间距
- 右上角有绿色 "Mini Mode" 标签
- 固定在 Header 下方
